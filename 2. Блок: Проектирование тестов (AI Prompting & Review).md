# Чек-лист MVP формы расчёта стоимости перевозки

# 1. Позитивные сценарии (Happy Path)
ID	                                  Проверка	                                                                       
- [ ] CHK-P01 Выбор валидного города отправления из автокомплита	                  
Почему критично: Без корректного города невозможно определить маршрут → расчет невозможен
- [ ] CHK-P02	 Выбор валидного города назначения из автокомплита	                    
Почему критично: Ошибка приводит к неверному маршруту и неверной стоимости
- [ ] CHK-P03	 Ввод валидного веса в пределах допустимого диапазона	                
Почему критично: Вес напрямую влияет на тариф → ошибка = финансовая ошибка
- [ ] CHK-P04	 Ввод валидного объема в пределах допустимого диапазона	              
Почему критично: Объем — ключевой тарифный параметр, влияет на цену
- [ ] CHK-P05	 Выбор даты погрузки (сегодня или будущая дата)	                      
Почему критично: Дата влияет на доступность маршрута и расчет стоимости
- [ ] CHK-P06	 Успешный расчет стоимости по валидным данным	                        
Почему критично: Основная бизнес-функция MVP — генерация цены
- [ ] CHK-P07	 Корректное отображение цены в рублях	                                
Почему критично: Ошибка отображения = бизнес-ошибка (финансовый ущерб/UX сбой)

# 2. Граничные значения (Boundary Value Analysis)
ID	                                  Проверка
- [ ] CHK-B01	Вес = 0 кг	Проверка бизнес-валидности (0 может быть запрещён)
Почему критично: риск некорректного тарифа
- [ ] CHK-B02	Вес = 20 000 кг (максимум)	Верхняя граница тарификации
Почему критично: риск переполнения или неверного расчета
- [ ] CHK-B03	Вес = 20 001 кг (выход за предел)	Проверка защиты бизнес-ограничения
Почему критично: предотвращение некорректных заказов
- [ ] CHK-B04	Объем = 0 м³
Почему критично: Возможна логическая ошибка тарификации (нулевой объем не должен проходить)
- [ ] CHK-B05	Объем = 82 м³ (максимум)
Почему критично: Верхняя граница расчета тарифа
- [ ] CHK-B06	Объем = 82.01 м³
Почему критично: Проверка точности ограничений (частая ошибка округления)
- [ ] CHK-B07	Дата = сегодня	Минимально допустимая дата
Почему критично: проверка логики доступности перевозки
- [ ] CHK-B08	Дата = вчера	Запрещённый сценарий
Почему критично: критично для планирования логистики

# 3. Негативные сценарии (Negative Testing)
ID	Проверка	Почему критично
- [ ] CHK-N01	Ввод произвольного текста вместо веса/объема
Почему критично: Ломает API, может вызвать 500 или некорректный расчет
- [ ] CHK-N02	Отрицательные значения веса/объема
Почему критично: Приводит к математически невозможному тарифу
- [ ] CHK-N03	Ввод слишком больших чисел (overflow)
Почему критично: Риск переполнения backend / SQL / float ошибок
- [ ] CHK-N04	Несуществующий город (если обход автокомплита возможен)
Почему критично: Маршрут не существует → ошибка расчета или падение API
- [ ] CHK-N05	Пустые обязательные поля
Почему критично: Прямое нарушение бизнес-валидации → некорректный запрос
- [ ] CHK-N06	Выбор одинакового города отправления и назначения
Почему критично: Логически некорректный маршрут → ошибка бизнес-логики
- [ ] CHK-N07	Прошедшая дата погрузки

- [ ] CHK-N08	Подмена запроса через DevTools (изменение payload)

- [ ] CHK-N09	SQL/XSS в городах (если возможен ручной ввод)
Почему критично: Критическая безопасность системы
- [ ] CHK-N10	Повторный быстрый клик по «Рассчитать стоимость»
Почему критично: Возможны дубли запросов и неконсистентные расчеты

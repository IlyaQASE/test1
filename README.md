Тестирование бизнес-процесса ЭТрН
Предпосылки:
На основании BPMN-схемы и описания формата электронного транспортного документа был составлен чек-лист проверки основного бизнес-процесса ЭТрН, ключевых валидаций и негативных сценариев.

Чек-лист:
    
   # 1. Проверка BPMN-схемы и бизнес-процесса
- [ ] Общая целостность процесса
- [ ] Процесс имеет единственную точку старта.
- [ ] Все ветки процесса завершаются конечным событием.
- [ ] Отсутствуют "висящие" переходы без продолжения.
- [ ] Для каждого шлюза (Gateway) определены все возможные исходы.
- [ ] Отсутствуют циклические переходы без условий выхода.
- [ ] Все события получения сообщений имеют соответствующие события отправки.
- [ ] Все переходы между участниками процесса логически обоснованы.
- [ ] Последовательность титулов соответствует бизнес-логике документооборота.

    - Проверка ролей участников
- [ ] Грузоотправитель может создать ЭТрН.
- [ ] Перевозчик получает документ после отправки.
- [ ] Перевозчик может подтвердить получение.
- [ ] Диспетчер получает данные о погрузке.
- [ ] Грузополучатель получает сведения по перевозке.
- [ ] Каждый участник имеет доступ только к своим операциям

    # 2. Создание ЭТрН
    Позитивные сценарии
- [ ] Создание ЭТрН с заполнением всех обязательных реквизитов.
- [ ] Создание ЭТрН с заполнением обязательных и необязательных реквизитов.
- [ ] Сохранение документа.
- [ ] Повторное открытие сохраненного документа.
- [ ] Редактирование сохраненного документа.
- [ ] Подписание документа ЭП.
- [ ] Отправка документа перевозчику.
    Негативные сценарии
- [ ] Попытка создания ЭТрН без обязательных реквизитов.
- [ ] Попытка подписания документа с ошибками валидации.
- [ ] Попытка отправки неподписанного документа.
- [ ] Попытка отправки документа с некорректными реквизитами.

  # 3. Первый титул
- [ ] Первый титул формируется после успешного создания ЭТрН.
- [ ] Первый титул содержит данные документа.
- [ ] Отображается корректный номер титула.
- [ ] Отображается корректная дата формирования.
- [ ] Титул доступен участникам процесса согласно ролям.

  # 4. Подтверждение получения перевозчиком
    Позитивные сценарии
- [ ] Перевозчик может подтвердить получение документа.
- [ ] После подтверждения изменяется статус документа.
- [ ] Действие фиксируется в истории документа.
    Негативные сценарии
- [ ] Повторное подтверждение невозможно.
- [ ] Невозможно подтвердить документ пользователем без соответствующей роли.

  # 5. Получение информации о погрузке
    Без замечаний
- [ ] Информация о погрузке успешно принимается.
- [ ] Данные передаются диспетчеру.
- [ ] Формируется второй титул.
    С замечаниями
- [ ] Доступен чекбокс "Замечания при погрузке".
- [ ] При установке чекбокса становятся доступны поля описания замечаний.
- [ ] Замечания сохраняются.
- [ ] Замечания отображаются в документе.
- [ ] Формируется второй титул с информацией о замечаниях.
    Негативные сценарии
- [ ] Установлен чекбокс замечаний, но описание не заполнено.
- [ ] Превышена максимальная длина описания замечаний.
- [ ] Попытка сохранить недопустимые символы в описании замечаний.

   # 6. Второй титул
- [ ] Формируется после обработки информации о погрузке.
- [ ] Содержит корректные сведения о погрузке.
- [ ] Отправляется грузоотправителю.
- [ ] Отправляется грузополучателю.
- [ ] Отображается в истории документа.

   # 7. Погрузка
- [ ] Возможно зафиксировать факт завершения погрузки.
- [ ] После завершения погрузки изменяется статус документа.
- [ ] После завершения погрузки формируется третий титул.
    Негативные проверки
- [ ] Невозможно начать перевозку до завершения погрузки.
- [ ] Невозможно сформировать третий титул до завершения погрузки.

   # 8. Третий титул
- [ ] Формируется после завершения погрузки.
- [ ] Отображает актуальное состояние перевозки.
- [ ] Корректно отображается всем участникам процесса.

   # 9. Перевозка
- [ ] Отображается статус перевозки.
- [ ] Отображаются сведения о грузе.
- [ ] Отображаются сведения о транспортном средстве.
- [ ] Отображаются сведения о водителе.

   # 10. Разгрузка
- [ ] Возможно зафиксировать факт разгрузки.
- [ ] После завершения разгрузки изменяется статус документа.
- [ ] После завершения разгрузки формируется четвертый титул.

   # 11. Четвертый титул
- [ ] Формируется после завершения разгрузки.
- [ ] Содержит итоговое состояние перевозки.
- [ ] Статус документа изменяется на "Завершен".
- [ ] Документ становится недоступным для редактирования.

   # 12. Проверка истории статусов
- [ ] Все изменения статусов сохраняются.
- [ ] Статусы отображаются в правильной последовательности.
- [ ] Для каждого статуса фиксируются дата и время изменения.
- [ ] Для каждого статуса фиксируется инициатор изменения.

   # 13. Проверки XSD и реквизитов
    ИНН
- [ ] ИНН из 10 цифр проходит валидацию.
- [ ] ИНН из 12 цифр проходит валидацию (если допускается схемой).
- [ ] ИНН менее допустимой длины отклоняется.
- [ ] ИНН более допустимой длины отклоняется.
- [ ] ИНН с буквами отклоняется.
- [ ] ИНН со спецсимволами отклоняется.
    КПП
- [ ] КПП из 9 символов проходит валидацию.
- [ ] КПП менее 9 символов отклоняется.
- [ ] КПП более 9 символов отклоняется.
    Дата
- [ ] Корректная дата проходит валидацию.
- [ ] 29.02 високосного года проходит валидацию.
- [ ] Некорректная дата отклоняется.
- [ ] Неверный формат даты отклоняется.
    Время
- [ ] Корректное время проходит валидацию.
- [ ] Время 23:59:59 проходит валидацию.
- [ ] Время 24:00:00 отклоняется.
- [ ] Некорректный формат времени отклоняется.
    Масса груза
- [ ] Допустимое значение массы сохраняется.
- [ ] Отрицательная масса отклоняется.
- [ ] Буквенное значение отклоняется.
- [ ] Значение с превышением допустимой точности отклоняется.

# Баг-репорты

Bug #1
Заголовок

Система позволяет сформировать третий титул без получения второго титула

Severity

Critical

Предусловия

Создана и отправлена ЭТрН.

Шаги воспроизведения
Создать ЭТрН.
Отправить документ перевозчику.
Не выполнять этап формирования второго титула.
Перейти к этапу завершения погрузки.
Сформировать третий титул.
Фактический результат

Третий титул успешно формируется.

Ожидаемый результат

Третий титул не должен формироваться до получения второго титула.

Комментарий

Нарушается последовательность юридически значимого документооборота.

Bug #2
Заголовок

Система позволяет сформировать второй титул при установленном признаке замечаний без заполнения описания несоответствий

Severity

Major

Предусловия

Документ находится на этапе обработки информации о погрузке.

Шаги воспроизведения
Открыть форму обработки информации о погрузке.
Установить чекбокс "Замечания при погрузке".
Не заполнять поля описания замечаний.
Сохранить документ.
Запустить формирование второго титула.
Фактический результат

Второй титул успешно формируется.

Ожидаемый результат

Система должна требовать заполнение данных о замечаниях и блокировать формирование титула.

Bug #3
Заголовок

После подтверждения получения перевозчиком статус ЭТрН не изменяется

Severity

Major

Предусловия

Перевозчик получил документ.

Шаги воспроизведения
Открыть полученную ЭТрН.
Нажать "Подтвердить получение".
Обновить карточку документа.
Фактический результат

Статус документа остается прежним.

Ожидаемый результат

Статус должен измениться на статус, соответствующий этапу подтверждения получения перевозчиком.

Комментарий

Пользователи не могут определить текущее состояние документа.
